Interpolação

A interpolação é feita por meio de duas chaves que envolvem uma variável definida no ts e apresenta o valor dessa variável na interface. Ela pode ser usada entre a abertura e o fechamento de tags ou em atributos de tags.

Ela realiza o one way data-binding, portanto só faz o caminho do ts para o html.

String renderizada com interpolação: {{ url }}

Resultado de expressão matemática (1+1) renderizada com interpolação: {{ 1 + 1 }}

Retorno de uma função renderizada com interpolação: {{ func() }}

Retorno de uma expressão boolean renderizada com interpolação: {{ true && false }}

Property binding

A property binding é feita somente em atributos de tags, o atributo deve estar entre colchetes para receber uma variável definida no ts, então o valor da variável será passado para o atributo.

Ela realiza o one way data-binding, portanto só faz o caminho do ts para o html.

Event binding

O event binding é feito envolvendo o nome de um evento entre parenteses em uma tag que se deseja escutar esse evento, quando disparado será acionado alguma função definida no ts

Two way data-binding

É a mistura entre o input property (ts para o html) e o event binding (html para o ts), pois e feito pela diretiva [(ngModel)] que envolte os colchetes e parênteses. Dessa forma temos o two way data-binding que consiste em permitir que o valor de uma variável possa ser alterada tanto no template quanto no ts.
